## Вернуть прогресс-бары

При рефакторинге на data-driven рендер я случайно убрал визуальные прогресс-бары — оставил только числа. Верну их, не трогая структуру знаний.

### Где вернуть

1. **Виджет «Индекс знания»** (`IndexWidget` в `src/components/KnowledgeBase.tsx`)
   - В кнопке-триггере: тонкая полоса с заливкой = `stats.percent` (актуальные / всего), рядом — число `NN%`.
   - В шапке drawer'а: та же полоса, шире, под подзаголовком «Состояние профиля».

2. **Карточки областей на главной** (`AreaCard`)
   - Под описанием, перед футером: прогресс-бар «актуальность области» = `current / total` карточек знаний этой области.
   - Слева подпись `NN%`, справа — `X из Y актуальны`.

3. **Левая колонка детальной страницы области** (`AreaView`, `np-area-left-item`)
   - У каждого пункта списка областей — мини-полоска прогресса под названием (как было раньше).

### Расчёт (моковый, из JSON)

Для каждой области:
```
total   = area.knowledge.length
current = area.knowledge.filter(k => k.freshness.code === "current").length
percent = round(current / total * 100)
```
Для виджета — сумма по всем областям (уже считается в `stats.percent`).

### Стили

Вернуть/добавить в `src/styles/norm-prototype.css`:
- `.np-progress` (трек, высота 4–6px, скругление, фон `#eef0f3`)

- Тоновые модификаторы: `--ok` (зелёный, ≥80%), `--warn` (жёлтый, 40–79%), `--low` (красный, <40%)
- Компактный вариант `.np-progress--sm` для левой колонки и виджета-кнопки

### Ограничения

- Не меняю данные, типы и структуру карточек знаний.
- Не трогаю табы, drawer-логику и навигацию.
- Только визуальное возвращение прогресс-баров + расчёт процентов из текущего JSON.
